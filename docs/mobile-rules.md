# Mobile Rules

## Principle

Mobile is a first-class experience.

## Requirements

- no awkward hero height
- no oversized spacing that forces endless scrolling
- no tiny text
- no weird hover-dependent UX
- no dense multi-column layouts
- no broken card hierarchy
- no animations that feel laggy or excessive

## Mobile section behavior

Hero:

- must remain compact and clear

Featured Work:

- cards stack cleanly
- screenshots remain meaningful
- spacing remains premium

More Projects:

- simple one-column list or compact cards

Experience:

- timeline must still be readable
- role hierarchy must remain obvious

Tech Stack:

- grouped clearly
- easy to scan
- no clutter

Snaggy:

- should use a clean mobile-friendly entry pattern
- likely bottom sheet or compact floating action

## Prohibition

Do not treat mobile as “desktop shrunk down.”
