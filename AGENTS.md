# AGENTS.md

This file defines the permanent operating rules for any AI agent working on this portfolio rebuild.

## Mission
Rebuild this portfolio into a premium, recruiter-focused, modern technical portfolio tailored to:
- AI / ML
- Cybersecurity
- Infrastructure / DevOps
- Full-stack engineering

The site must feel:
- clean
- premium
- minimal
- high-signal
- mobile-polished
- technically credible

## Non-Negotiables
Always follow the docs in `/docs`.
If any implementation conflicts with the docs, the docs win.

Read before doing major work:
- /docs/vision.md
- /docs/brand.md
- /docs/information-architecture.md
- /docs/design-system.md
- /docs/content-strategy.md
- /docs/project-strategy.md
- /docs/experience-strategy.md
- /docs/mobile-rules.md
- /docs/milestones.md
- /docs/definition-of-done.md

## Primary Rule
This site is for recruiters and hiring managers first.
Not for showing off fancy effects.
Not for maximizing animation.
Not for experimental design.

## Build Principles
1. Clarity over creativity
2. Premium feel over flashy effects
3. Scanability over volume
4. Strong hierarchy over decorative complexity
5. Mobile quality is mandatory
6. Every section must earn its place
7. Fewer, better sections
8. Maintainable code and reusable components
9. Performance matters
10. Restraint is part of quality

## Preserve
These ideas should remain, but be modernized:
- terminal-inspired `>` prompt
- strong hero intro
- Featured Projects
- More Projects
- Experience timeline
- simplified Tech Stack
- Snaggy AI as secondary enhancement

## Remove / Avoid
Never add back:
- loading screen
- delayed intro before real content
- neural particle backgrounds
- skill percentages
- tiny thumbnails in experience
- cluttered layouts
- long dense paragraphs
- awkward mobile UX
- gratuitous glow and cyberpunk gimmicks

## Content Rules
- recruiters skim
- no long paragraphs unless necessary
- section intros must be short
- project cards must be concise
- experience must be highly scannable
- tech stack must be grouped, not rated

## Project Rules
Featured projects are:
1. KnowledgeMesh
2. SentinelStack AI
3. PathToOffer
4. StudyNFlow

KnowledgeMesh is the hero project.

There must also be a More Projects section.

## Experience Rules
Show:
- CIRA
- AmanahTech

Timeline stays, but must be redesigned to be clean, premium, and readable.
No messy visual treatment.
No tiny images.
Text must be reduced to high-signal highlights.

## Snaggy Rules
Snaggy AI stays, but as a tasteful secondary feature.
It must not dominate the experience.
It should support discoverability, not hijack the site.

## Workflow Rules
Work in milestones defined in `/docs/milestones.md`.
At the end of each milestone:
- summarize completed work
- identify gaps
- verify alignment with docs
- recommend the next milestone only

## Decision Filter
Before adding anything, ask:
1. Does this improve recruiter clarity?
2. Does this improve perceived quality?
3. Does this make the site easier to scan?
4. Does this improve mobile?
5. Is this consistent with the premium technical brand?
6. Is this necessary?

If the answer is no, do not add it.

## Definition of Success
The finished portfolio should make a recruiter think:
“This candidate builds real systems and has strong taste.”
